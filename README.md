# 🕹️ Pixel Gacha (像素扭蛋)

一个纯原生 JavaScript 驱动的像素风交互实验。这不仅是一个简单的点击效果，更是一个针对移动端性能与音效延迟优化的**互动原型 (Interactive Prototype)**。

---

## 🌟 核心亮点

* **👾 纯正像素美学**：完全使用 CSS `box-shadow` 栈和 `canvas` 像素化渲染，无需外部图片资源，实现极简的视觉风格。
* **🧪 独立物理模拟**：自主实现的圆体碰撞检测（Circle Collision）与重力感应逻辑，支持指针/触摸动量实时交互。
* **🚀 零延迟音效引擎 (Web Audio API)**：
    * **秒开体验**：针对移动端 Safari/Chrome 深度优化，将音效预载入内存缓冲区，彻底解决 `new Audio()` 导致的 5 秒播放延迟。
    * **沉浸反馈**：高频碰撞时自动触发密集 `pop` 音效，不弹系统喇叭，不触发媒体控制栏。
* **🛡️ 全环境兼容机制**：自动识别浏览器限制，在 Web Audio 受阻时智能降级至传统音频模式，确保电脑与手机端均有完美表现。

---

## 🛠️ 开发日志：如何干掉那 5 秒延迟？

在开发这个 Demo 时，最大的挑战是移动端浏览器的**省电策略**。

1. **痛点**：普通 HTML5 音频在用户点击后才发起网络请求，导致声音和画面严重脱节（延迟 1-5s）。
2. **方案**：改用 **Web Audio API**。我们在页面初始化时通过二进制流（ArrayBuffer）预取音频，点击时直接从 CPU 调用内存数据。
3. **结果**：响应速度从“秒级”提升到了“毫秒级”，真正实现了丝滑的交互反馈。

## 🎮 玩法指南

- **点击/长按标题**：喷射随机色彩的像素扭蛋。
- **滑动/拖拽鼠标**：扰乱重力场，与已有的扭蛋进行实时物理互动。
- **双端自适应**：电脑端支持高上限（80+）扭蛋共存，手机端自动限制数量以确保帧率稳定。

## 🏗️ 快速本地调试

由于涉及到 `fetch` 请求内存音频，请勿直接双击 `.html` 文件打开（会导致跨域拦截）。

推荐方式：
1. 使用 VS Code 插件 **Live Server** 启动。
2. 或使用 Python 快速搭建：`python -m http.server 8000`
